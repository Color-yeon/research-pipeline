# 논문 교차 비교 스킬

## 0단계: 선행 조건 검사

이 스킬을 실행하기 전에 **반드시** 아래 명령을 Bash 도구로 실행하라.

```bash
node scripts/lib/pipeline-guard.mjs research-compare
```

- exit 0 → 통과. 다음 단계로 진행한다.
- exit code 가 0 이 아니면 → stderr 의 사유를 사용자에게 그대로 보고하고
  **실행을 즉시 중단**하라. 필요한 선행 스킬(예: `/research-search`)을
  먼저 수행해야 한다.

Claude Code 에서는 `.claude/settings.json` 의 PreToolUse 훅이 같은 검사를
이벤트 수준에서 추가로 수행한다. 하지만 Codex·Gemini 경로에서는 이 명령이
유일한 방어선이다.

## 개요

2개 이상의 논문을 체계적으로 교차 비교하여, 방법론/결과/주장의 합의점과 불일치점을 분석하고, 연구 갭을 식별한다. 비교 결과는 구조화된 대조표와 분석 리포트로 출력한다.

## 인자

- `$ARGUMENTS`: DOI 목록 (2~N개, 공백 또는 쉼표로 구분) 또는 findings 파일 경로

### 입력 예시
```
# DOI 목록
10.1234/abc, 10.5678/def, 10.9012/ghi

# findings 파일 경로
findings/keyword_combination_1.md

# 혼합
10.1234/abc findings/keyword_combination_1.md
```

## 비교 축

4개의 비교 축을 기준으로 논문 간 교차 비교를 수행한다:

| 비교 축 | 비교 항목 |
|---------|----------|
| **방법론** | 연구 설계, 샘플 크기, 측정 도구, 통계 방법, 데이터 수집 방식 |
| **결과** | 핵심 발견, 효과 크기, 통계적 유의성, 부정적 결과 포함 여부 |
| **주장** | 결론의 강도, 일반화 범위, 실무적 시사점, 이론적 기여 |
| **한계점** | 인정된 한계, 외적 타당도, 재현 가능성, 시간적 한계 |

## 절차

### 1단계: 증거 카드 로드

1. **DOI가 주어진 경우**:
   - `findings/` 디렉토리에서 해당 DOI의 증거 카드 검색
   - 기존 증거 카드가 있으면 로드
   - 없으면 WebSearch + Semantic Scholar에서 기본 정보 수집

2. **findings 파일이 주어진 경우**:
   - 파일을 읽어서 모든 논문의 증거 카드 추출
   - DOI 기준으로 중복 제거

3. **비교 대상 확인**:
   - 최소 2개 이상의 논문이 로드되었는지 확인
   - 각 논문의 기본 정보(제목, 저자, 연도, 저널, DOI) 확인

### 2단계: 비교 축 기준 대조표 생성

각 비교 축에 대해 구조화된 대조표를 생성한다.

#### 방법론 대조표
| 항목 | 논문A | 논문B | 논문C | ... |
|------|-------|-------|-------|-----|
| 연구 설계 | | | | |
| 샘플 크기 | | | | |
| 대상 집단 | | | | |
| 측정 도구 | | | | |
| 통계 방법 | | | | |
| 데이터 수집 기간 | | | | |

| 부정적 결과 | | | | |

### 3단계: 합의점/불일치점 분석

#### 합의점 (Consensus)
- 모든(또는 대다수) 논문이 동의하는 발견/주장 식별
- 합의의 강도 평가 (강한 합의 / 약한 합의 / 다수결)
- 합의 사항의 증거 수준 평가

#### 불일치점 (Disagreement)
- 논문 간 상충하는 발견/주장 식별
- 불일치의 원인 분석:
  - 방법론 차이에 의한 불일치
  - 표본 차이에 의한 불일치
  - 측정 도구 차이에 의한 불일치
  - 해석 차이에 의한 불일치
  - 시간적 차이에 의한 불일치
- 어느 쪽이 더 신뢰할 수 있는지 근거 제시

### 4단계: 연구 갭 식별

- 모든 논문이 다루지 않은 영역 식별
- 불일치를 해소하기 위해 필요한 연구
- 합의된 발견을 확장하기 위한 연구 방향
- 방법론적 갭 (더 엄밀한 설계가 필요한 부분)
- 이론적 갭 (설명되지 않은 현상)

## 종료 단계: 체크포인트 저장

스킬 실행을 완료하기 직전에 **반드시** 아래 명령을 Bash 도구로 실행하라.

```bash
node scripts/lib/checkpoint.mjs
```

findings/ 전체 상태를 `findings/_checkpoint.json` 에 기록하여 다음 태스크
(또는 컨텍스트 압축 이후) 가 진행 상황을 이어갈 수 있게 한다. 실패해도
스킬은 정상 종료로 간주한다(베스트 에포트).

Claude Code 에서는 `.claude/settings.json` 의 PreCompact 훅이 같은 일을
컨텍스트 압축 직전에 수행한다. 하지만 Codex·Gemini 경로에서는 이 명령이
유일한 체크포인트 경로다.
