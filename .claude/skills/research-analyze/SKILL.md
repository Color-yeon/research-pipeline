# 통합 분석 및 커버리지 감사

## 0단계: 선행 조건 검사

이 스킬을 실행하기 전에 **반드시** 아래 명령을 Bash 도구로 실행하라.

```bash
node scripts/lib/pipeline-guard.mjs research-analyze
```

- exit 0 → 통과. 다음 단계로 진행한다.
- exit code 가 0 이 아니면 → stderr 의 사유를 사용자에게 그대로 보고하고
  **실행을 즉시 중단**하라. 필요한 선행 스킬(예: `/research-search`)을
  먼저 수행해야 한다.

Claude Code 에서는 `.claude/settings.json` 의 PreToolUse 훅이 같은 검사를
이벤트 수준에서 추가로 수행한다. 하지만 Codex 경로에서는 이 명령이
유일한 방어선이다.

## 입력 ($ARGUMENTS)

`$ARGUMENTS`는 선택적이다:
- **미지정 또는 `"all"`**: `findings/` 디렉토리 전체를 대상으로 통합 분석 수행 (기본 동작)
- **특정 파일 경로**: `"findings/keyword_combination_1.md"` — 해당 파일만 대상으로 분석

대부분의 경우 전체 findings를 대상으로 실행한다.

## 절차

### 1단계: findings/ 전체 읽기

1. `findings/` 디렉토리의 **모든** 마크다운 파일을 읽는다.
2. 각 파일에서 증거 카드를 파싱하여 전체 논문 목록을 구축한다.
3. 추출 대상 정보:
   - 논문 제목, 저자, 연도, 저널, DOI
   - 인용수, 방법론, 핵심 발견, 한계점, 신뢰도
   - 관련 키워드 조합
   - 출처 파일명 (어느 findings 파일에서 수집되었는가)

### 2단계: 논문 목록 통합 및 중복 제거

1. DOI 기준으로 전체 목록에서 중복을 제거한다.
2. DOI가 없는 논문은 제목+저자로 중복 판정한다.
3. 통합 논문 목록을 작성한다:
   ```markdown
   ## 통합 논문 목록 (중복 제거 후)
   | # | 제목 | 저자 | 연도 | 저널 | DOI | 신뢰도 | 출처 |
   ```

### 3단계: 논문 간 관계 분석

1. **인용 네트워크**: 논문 간 인용 관계를 매핑한다.
   - 어떤 논문이 가장 많이 인용되는가? (허브 논문)
   - 인용 클러스터가 존재하는가?
2. **시간적 흐름**: 연도별 논문 분포를 분석한다.
   - 연구가 활발한 시기와 정체기 식별
   - 패러다임 전환 시점 식별
3. **방법론 분포**: 사용된 연구 방법론의 분포를 분석한다.
   - 지배적인 방법론은 무엇인가?
   - 새로운 방법론 트렌드가 있는가?
4. **결과 일치/불일치**: 유사 주제 논문 간 결과 비교
   - 일관된 결과를 보이는가?
   - 상충하는 결과가 있다면 원인은 무엇인가?

### 4단계: 연구 갭 식별

다음 관점에서 연구 갭을 식별한다:
1. **주제 갭**: 탐구되지 않은 하위 주제나 변수 조합
2. **방법론 갭**: 특정 방법론이 적용되지 않은 영역
3. **대상 갭**: 특정 인구/조건/환경에서의 연구 부재
4. **시간 갭**: 최근 N년간 업데이트되지 않은 영역
5. **재현 갭**: 한 번만 보고되고 재현되지 않은 결과

### 5단계: 커버리지 감사

**이 단계가 가장 중요하다. 빠진 논문이 있어서는 안 된다.**

1. **추가 검색으로 검증**:
   - 통합 목록의 핵심 키워드로 WebSearch 재검색
   - OpenAlex, Semantic Scholar, Google Scholar에서 교차 검증
   - 수집되지 않은 관련 논문이 발견되면 `[감사에서 추가 발견]` 태그와 함께 기록

2. **핵심 저자 기반 검증**:
   - 주요 저자의 다른 논문 목록을 확인
   - 해당 저자의 출판 목록에서 빠진 관련 논문이 있는지 확인

3. **핵심 저널 기반 검증**:
   - 관련 논문이 많이 게재된 저널에서 추가 검색
   - 해당 저널의 최근 호(issue)를 브라우징

4. **감사 결과 기록**:
   ```markdown
   ## 커버리지 감사 결과
   - 감사 전 총 논문 수: N개
   - 추가 발견 논문 수: M개
   - 감사 후 총 논문 수: N+M개
   - 검증 방법: [사용한 검증 방법 목록]
   - 자체 점검: 더 찾을 수 있는 논문이 남아있는가? [예/아니오 + 근거]
   ```

### 6단계: 핵심 저자/랩 정리

1. 논문 수 기준 상위 저자 목록 작성
2. 각 저자의 소속 기관 확인
3. 연구 그룹/랩 간 협력 관계 매핑
4. 해당 분야의 선도 그룹 식별

## 출력

### findings/integrated_analysis.md
- 통합 논문 목록 (중복 제거 후)
- 논문 간 관계 분석 (인용 네트워크, 시간적 흐름, 방법론 분포)
- 결과 일치/불일치 분석
- 연구 갭 목록
- 향후 연구 방향 제안
- `## DOI 목록` 섹션 (하단)

### findings/audit_report.md
- 커버리지 감사 결과
- 추가 발견 논문 목록 (증거 카드 형식)
- 감사 통계 요약
- 검증 방법 상세

### findings/authors_labs.md
- 핵심 저자 목록 (논문 수, 인용수, 소속)
- 핵심 연구 그룹/랩 목록
- 협력 관계 매핑
- 연락처 정보 (가능한 경우)

## 종료 단계: 체크포인트 저장

스킬 실행을 완료하기 직전에 **반드시** 아래 명령을 Bash 도구로 실행하라.

```bash
node scripts/lib/checkpoint.mjs
```

findings/ 전체 상태를 `findings/_checkpoint.json` 에 기록하여 다음 태스크
(또는 컨텍스트 압축 이후) 가 진행 상황을 이어갈 수 있게 한다. 실패해도
스킬은 정상 종료로 간주한다(베스트 에포트).

Claude Code 에서는 `.claude/settings.json` 의 PreCompact 훅이 같은 일을
컨텍스트 압축 직전에 수행한다. 하지만 Codex 경로에서는 이 명령이
유일한 체크포인트 경로다.
