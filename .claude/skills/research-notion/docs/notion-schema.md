# 노션 DB 스키마 정의

## 1. 문헌 데이터베이스 속성 매핑 규칙

### 주요 속성

| 속성명 | 노션 타입 | findings 카드 필드 | 비고 |
|--------|----------|-------------------|------|
| 논문 제목 | TITLE | `### [논문 제목]` | 필수. 증거 카드의 제목에서 추출 |
| 저자 | RICH_TEXT | `저자` 행 | "성1, 성2, ..." 형식 |
| 연도 | NUMBER | `연도` 행 | 정수형 (예: 2024) |
| 저널 | RICH_TEXT | `저널` 행 | 저널 전체 이름 사용 |
| DOI | URL | `DOI` 행 | `https://doi.org/` 접두사 포함 |
| 인용수 | NUMBER | `인용수` 행 | 정수형 |
| 방법론 | RICH_TEXT | `방법론` 행 | 주요 방법론 간략 기술 |
| 신뢰도 | SELECT | `신뢰도` 행 | 높음(green)/보통(yellow)/낮음(red) |
| 키워드 조합 | MULTI_SELECT | `관련 키워드 조합` 행 | 동적 옵션 추가 |
| 핵심 발견 | RICH_TEXT | `핵심 발견` 행 | 200자 이내 요약 |
| 한계점 | RICH_TEXT | `한계점` 행 | 200자 이내 요약 |
| 전문 확인 | CHECKBOX | 전문 읽기 여부 | `[전문 미확인]` 태그가 없으면 체크 |
| 출처 파일 | RICH_TEXT | 파일명 | findings 내 원본 파일명 |

### 선택적 속성 (필요 시 추가)

| 속성명 | 노션 타입 | 용도 |
|--------|----------|------|
| 방법론 등급 | SELECT | methods_critique.md의 A~F 등급 |
| Snowball 깊이 | NUMBER | snowball 추적에서 발견된 깊이 |
| 연구 유형 | SELECT | RCT/코호트/리뷰/메타분석 등 |
| 국가 | MULTI_SELECT | 연구 수행 국가 |
| 비고 | RICH_TEXT | 추가 메모 |

## 2. 속성 매핑 시 특수 처리

### DOI 처리
- 정상 DOI: `https://doi.org/10.xxxx/yyyy` 형태로 URL 속성에 저장
- `[DOI 미검증]`: DOI 필드에 빈 값, 비고에 `"DOI 미검증"` 기재
- `DOI: 미확인`: DOI 필드에 빈 값, 비고에 `"DOI 미확인"` 기재

### 신뢰도 매핑
| findings 값 | 노션 SELECT 값 | 색상 |
|-------------|----------------|------|
| 높음 | 높음 | green |
| 보통 | 보통 | yellow |
| 낮음 | 낮음 | red |
| `[LOW-CREDIBILITY]` 태그 | 낮음 | red |

### 인용수 처리
- 숫자만 추출 (예: `"약 150회"` → `150`)
- `"N/A"` 또는 미기재 → `null` (빈 값)

### 키워드 조합 옵션
- MULTI_SELECT 옵션은 findings 파일명에서 자동 추출
- 예: `findings/keyword_combination_1.md` → `"keyword_combination_1"` 옵션 추가
- Snowball 파일: `"snowball_depth1"`, `"snowball_depth2"` 등으로 추가

## 3. 페이지 구조 (개별 논문 페이지)

각 논문 페이지의 본문은 다음 구조를 따른다:

```markdown
## 증거 카드

| 항목 | 내용 |
|------|------|
| 저자 | ... |
| 연도 | ... |
| 저널 | ... |
| DOI | ... |
| 인용수 | ... |
| 방법론 | ... |
| 핵심 발견 | ... |
| 한계점 | ... |
| 신뢰도 | ... |
| 관련 키워드 조합 | ... |

## 상세 요약

[논문 내용의 상세 요약 — 초록 수준 이상]

## 방법론 메모

[실험 설계에 대한 메모 (methods_critique.md가 있으면 해당 내용 포함)]

## 인용 관계

[이 논문을 인용한 수집 내 논문 목록 / 이 논문이 인용한 수집 내 논문 목록]
```

## 5. 배치 처리 규칙

### 페이지 생성 배치
- 한 번의 `notion-create-pages` 호출에 최대 100개 페이지
- 100개 초과 시 배치 분할
- 각 배치 사이 오류 확인

### 오류 처리
- 페이지 생성 실패 시 해당 논문을 `findings/notion_failed.txt`에 기록
- 실패 사유 함께 기록 (속성 형식 오류, API 제한 등)
- 전체 프로세스를 중단하지 않고 나머지 논문은 계속 진행

### 중복 방지
- 노션에 이미 동일 DOI의 논문 페이지가 있는지 확인 (notion-search 활용)
- 중복 발견 시 기존 페이지 업데이트 또는 스킵
