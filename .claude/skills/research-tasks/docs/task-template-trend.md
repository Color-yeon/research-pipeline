# Trend 모드 태스크 시퀀스 상세

동향탐구(Trend) 모드의 각 Stage에 대한 상세 설명과 의존성 규칙.

---

## Stage 1: 리뷰 논문 탐색

### 목적
해당 분야의 최신 리뷰 논문과 메타분석을 검색하고, 가장 적합한 리뷰 논문을 선정한다.

### 태스크 내용
1. 다중 소스 검색 (WebSearch, OpenAlex, Semantic Scholar, Google Scholar)
   - 쿼리에 `review`, `systematic review`, `meta-analysis`, `survey` 등 추가
   - 최근 5년 이내 논문 우선
2. 발견된 리뷰 논문에 대해 증거 카드 작성
3. 리뷰 논문 선정 기준에 따라 순위 매기기:
   - **인용수** — 높을수록 우선
   - **Impact Factor** — 높을수록 우선
   - **최신성** — 최근 출판일수록 우선
   - **포괄성** — 참고문헌 수가 많을수록 우선
4. 상위 3~5편의 리뷰 논문 선정
5. **사기성/predatory 저널의 논문은 완전 제외** (수집도 하지 않음)

### 의존성
- 없음 (시작 태스크)

### 예상 출력
- `findings/review_papers.md` — 선정된 리뷰 논문 증거 카드

### 선정 기준 세부

| 우선순위 | 기준 | 비중 |
|---------|------|------|
| 1 | 최근 3년 이내 + IF 상위 저널 | 40% |
| 2 | 인용수 상위 | 30% |
| 3 | 참고문헌 100개 이상 | 20% |
| 4 | 체계적 리뷰(PRISMA 준수) | 10% |

---

## Stage 2: 참고문헌 전수조사

### 목적
선정된 리뷰 논문의 참고문헌을 **하나도 빠짐없이** 전수 수집한다.

### 태스크 분할
- 선정된 리뷰 논문 **각각에 대해 별도 태스크** 생성
- 예: 리뷰 3편 선정 → 태스크 3개

### 각 태스크의 내용
1. Playwright MCP(`browser_run_code`)로 리뷰 논문 전문에 접근
2. References 섹션의 **모든** 참고문헌을 추출
3. 각 참고문헌에 대해:
   - DOI 확인 (WebSearch 또는 Semantic Scholar)
   - 기본 메타데이터 수집 (저자, 연도, 저널, 제목)
   - 연구 주제와의 관련성 판단
4. 관련 논문에 대해 증거 카드 작성
5. 비관련 논문도 목록에는 기록 (`[비관련]` 태그)
6. **절대로 참고문헌을 건너뛰지 않는다**

### 의존성
- Stage 1 완료 후 실행

### 예상 출력
- `findings/review_{리뷰논문명}_refs.md` — 리뷰별 참고문헌 분석
- 각 파일에 다음 포함:
  ```
  ## 참고문헌 전수조사 현황
  - 리뷰 논문: [제목]
  - 총 참고문헌 수: N개
  - 관련 논문: M개 (증거 카드 작성)
  - 비관련 논문: K개 (목록만 기록)
  - 접근 불가: L개
  - 누락: 0개 (전수조사 완료)
  ```

### 주의사항
- Trend 모드에서는 **참고문헌 전수조사가 핵심**이다.
- "하나도 빠지면 안 된다"는 원칙을 철저히 준수한다.
- 참고문헌이 100개 이상이어도 전부 처리한다.
- 시간이 오래 걸려도 괜찮다.

## Stage 3: 최신 트렌드 분석

### 목적
최근 2~3년 이내 논문을 집중 분석하여 연구 트렌드를 도출한다.

### 태스크 내용
1. Stage 2에서 수집된 논문 중 최근 2~3년 논문 필터링
2. 추가 검색: 최신 논문 전용 쿼리 (최근 연도 필터)
   - 프리프린트 서버(arXiv, bioRxiv, medRxiv) 포함
3. 시간순 트렌드 분석:
   - 신규 등장 키워드/개념
   - 방법론 변화 (새로운 기법 도입)
   - 연구 규모 변화 (샘플 크기, 다기관 연구 등)
   - 연구 초점 변화 (어떤 하위 주제가 주목받는가?)
4. 향후 예상 트렌드 도출

### 의존성
- Stage 2 완료 후 실행

### 예상 출력
- `findings/trend_latest.md` — 최신 트렌드 분석 보고서
- 보고서 구조:
  ```markdown
  ## 최신 트렌드 분석

  ### 1. 신규 등장 개념/키워드
  - ...

  ### 2. 방법론 트렌드
  - ...

  ### 3. 연구 규모 변화
  - ...

  ### 4. 연구 초점 이동
  - ...

  ### 5. 향후 예상 트렌드
  - ...

  ### 6. 핵심 최신 논문 (2024~현재)
  [증거 카드 목록]
  ```

## Stage 4: 감사

### 목적
전체 커버리지를 감사하고, 빠진 논문이 없는지 최종 검증한다.

### 태스크 내용
1. `research-analyze` 스킬의 감사 기능 실행
2. 수집된 전체 논문 목록을 다른 소스와 교차 검증
3. 핵심 저자의 출판 목록 확인
4. 리뷰 논문 간 참고문헌 교집합/차집합 분석
   - 리뷰 A에만 있는 논문, 리뷰 B에만 있는 논문
   - 어떤 리뷰에도 없지만 관련된 논문
5. 감사 보고서 작성

### 의존성
- Stage 3 완료 후 실행

### 예상 출력
- `findings/audit_report.md` — 감사 보고서
- `findings/authors_labs.md` — 핵심 저자/랩 목록

---

## Stage 5: 노션 기록

### 목적
모든 결과를 노션 데이터베이스로 구조화하여 기록한다.

### 태스크 내용
1. `research-notion` 스킬 실행
2. 인자: `all`
3. 문헌 DB 생성 + 논문별 페이지 생성
4. 리뷰별 참고문헌 분석 결과도 별도 페이지로 생성
5. 트렌드 분석 결과를 별도 페이지로 생성
6. DOI 목록 페이지 생성

### 의존성
- Stage 4 완료 후 실행

### 예상 출력
- 노션 데이터베이스 + 페이지
- `findings/notion_pages.txt`

```
                              └─ 리뷰 C refs ──┘

                                                │

```

## Trend 모드 특별 규칙 (재확인)

1. **사기성/predatory 논문 완전 제외**: 수집도 하지 않는다.
2. **리뷰 논문 선정**: 인용수 + IF + 최신성으로 판단한다.
3. **참고문헌 전수조사**: 하나도 빠지면 안 된다. 이것이 가장 중요하다.
4. **최신성 우선**: 최근 논문에 더 많은 가중치를 부여한다.
