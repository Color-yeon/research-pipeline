# 방법론 평가 체크리스트

논문의 Methods/Materials 섹션을 분석할 때 아래 체크리스트를 항목별로 점검한다.

---

## 1. 연구 설계 (Study Design)

### 점검 항목
- [ ] 연구 설계 유형이 명시되어 있는가?
- [ ] 연구 질문에 적합한 설계인가?
- [ ] 탐색적(exploratory) vs 확증적(confirmatory) 연구 구분이 명확한가?
- [ ] 연구 기간(duration)이 적절한가?
- [ ] 전향적(prospective) vs 후향적(retrospective) 여부가 기재되었는가?

### 설계별 추가 점검
| 설계 유형 | 추가 확인 사항 |
|-----------|---------------|
| RCT | 무작위 배정 방법, 배정 은닉(allocation concealment) |
| 코호트 | 추적 기간, 탈락률, 교란 변수 통제 |
| 사례-대조 | 사례/대조 정의, 매칭 기준, 회상 편향 통제 |
| 횡단면 | 시간적 인과관계 한계 인지, 응답률 |
| 메타분석 | 검색 전략, 포함 기준, 이질성 평가, 출판 편향 검정 |
| 동물실험 | ARRIVE 가이드라인 준수, 종/계통, 사육 조건 |
| in vitro | 세포주 인증, 계대 수, 오염 검사 |

## 2. 샘플 (Sample/Participants)

### 점검 항목
- [ ] 샘플 크기(n)가 명시되어 있는가?
- [ ] 검정력 분석(power analysis)이 사전에 수행되었는가?
  - 효과 크기 추정 근거는 무엇인가?
  - 유의수준(alpha)과 검정력(1-beta)은 얼마인가?
- [ ] 포함 기준(inclusion criteria)이 명확한가?
- [ ] 제외 기준(exclusion criteria)이 명확한가?
- [ ] 모집 방법(recruitment)이 기술되어 있는가?
- [ ] 탈락/미완료(dropout/attrition) 비율이 보고되었는가?
- [ ] ITT(intention-to-treat) vs PP(per-protocol) 분석 구분이 있는가?

### 샘플 크기 적절성 판단 기준
| 연구 유형 | 일반적 최소 권장 |
|-----------|-----------------|
| 파일럿 연구 | 군당 12~30명 |
| RCT | 군당 30명 이상 (검정력 분석 기반) |
| 설문 연구 | 100명 이상 |
| 질적 연구 | 포화(saturation) 도달까지 |
| 동물실험 | 군당 6~10마리 (3R 원칙 고려) |

## 3. 통계 방법 (Statistical Methods)

### 점검 항목
- [ ] 사용된 통계 검정이 명시되어 있는가?
- [ ] 데이터 분포 가정(정규성 등)이 확인되었는가?
- [ ] 데이터 유형에 맞는 검정이 사용되었는가?
  - 연속형: t-test, ANOVA, 회귀분석 등
  - 범주형: 카이제곱, Fisher 정확 검정 등
  - 시간-사건: 생존 분석, Cox 회귀 등
- [ ] 다중 비교 보정이 수행되었는가?
  - Bonferroni, Holm, FDR(Benjamini-Hochberg) 등
- [ ] 효과 크기(effect size)가 보고되었는가?
  - Cohen's d, odds ratio, hazard ratio, R-squared 등
- [ ] 신뢰구간(confidence interval)이 제시되었는가?
- [ ] 결측치(missing data) 처리 방법이 기술되었는가?
- [ ] 사용한 통계 소프트웨어와 버전이 기재되었는가?

### 흔한 통계적 문제
- p-hacking 의심: 다수의 비교 중 유의한 것만 보고
- HARKing: 결과를 본 후 가설을 사후적으로 설정
- 낮은 검정력: 샘플 부족으로 제2종 오류 가능성
- 부적절한 검정: 비모수 데이터에 모수 검정 적용

## 4. 대조군 (Controls)

### 점검 항목
- [ ] 적절한 대조군이 설정되어 있는가?
- [ ] 음성 대조군(negative control)이 있는가?
- [ ] 양성 대조군(positive control)이 있는가?
- [ ] 위약(placebo) 대조가 사용되었는가? (해당 시)
- [ ] 대조군과 실험군의 기저 특성(baseline characteristics)이 비교되었는가?
- [ ] 교란 변수(confounders)가 식별되고 통제되었는가?

### 대조군 적절성 평가
| 등급 | 기준 |
|------|------|
| 우수 | 양성+음성 대조군 모두 포함, 기저 특성 균형 |
| 적절 | 기본 대조군 포함, 주요 교란 변수 통제 |
| 부족 | 대조군 불충분 또는 교란 변수 미통제 |
| 미달 | 대조군 없음 |

## 5. 블라인딩 (Blinding)

### 점검 항목
- [ ] 블라인딩 수준이 명시되어 있는가?
  - 단일 블라인드(참가자만)
  - 이중 블라인드(참가자 + 연구자)
  - 삼중 블라인드(참가자 + 연구자 + 분석자)
- [ ] 블라인딩이 유지되었는지 확인하는 절차가 있는가?
- [ ] 블라인딩 불가능한 경우 그 이유와 대안이 기술되었는가?
- [ ] 결과 평가자(outcome assessor)의 블라인딩 여부는?

## 6. 재현 가능성 (Reproducibility)

### 점검 항목
- [ ] 프로토콜이 다른 연구자가 재현할 수 있을 정도로 상세한가?
- [ ] 시약, 장비, 소프트웨어의 제조사/버전/카탈로그 번호가 기재되었는가?
- [ ] 실험 조건(온도, 습도, 시간 등)이 명시되었는가?
- [ ] 원시 데이터(raw data)가 공개되었는가?
  - 저장소 이름과 접근 코드(accession number) 기재 여부
- [ ] 분석 코드/스크립트가 공개되었는가?
  - GitHub, Zenodo 등 저장소 링크 여부
- [ ] 사전 등록(pre-registration) 여부
  - ClinicalTrials.gov, OSF, PROSPERO 등

### 재현 가능성 등급
| 등급 | 기준 |
|------|------|
| 높음 | 프로토콜 상세 + 데이터 공개 + 코드 공개 + 사전 등록 |
| 보통 | 프로토콜 상세 + 데이터 또는 코드 공개 |
| 낮음 | 프로토콜 불충분 또는 데이터/코드 비공개 |
| 매우 낮음 | 핵심 절차 누락, 재현 불가능 |

## 7. 데이터 가용성 (Data Availability)

### 점검 항목
- [ ] 데이터 가용성 성명(Data Availability Statement)이 있는가?
- [ ] 원시 데이터 접근 방법이 명시되었는가?
- [ ] 데이터 공유 제한 사유가 합리적인가? (개인정보, 윤리 등)
- [ ] 보충 자료(Supplementary Materials)가 제공되는가?

## 8. 윤리적 고려사항

### 점검 항목 (해당 시)
- [ ] IRB/윤리위원회 승인 번호가 기재되었는가?
- [ ] 사전 동의(informed consent) 절차가 기술되었는가?
- [ ] 동물실험인 경우 IACUC 승인이 있는가?
- [ ] 이해충돌(conflict of interest) 선언이 있는가?
- [ ] 연구 지원(funding) 출처가 명시되었는가?
